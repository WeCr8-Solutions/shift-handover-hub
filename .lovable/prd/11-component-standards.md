# PRD 11 — Component Creation & Edit Standards

**Status**: ✅ Active  
**Last Updated**: 2026-03-08  
**Rule**: MUST be reviewed before creating, editing, or modifying any component.

---

## 1. Imports Checklist

Every component file must use the correct import sources. **Never guess — verify.**

| Need | Import From | ❌ Never |
|------|------------|----------|
| `cn()` | `@/lib/utils` | Don't redefine or inline `clsx`/`twMerge` |
| UI primitives (Button, Input, Dialog…) | `@/components/ui/*` | Don't copy-paste raw Radix |
| Supabase client | `@/integrations/supabase/client` | Don't create new clients |
| DB types | `@/integrations/supabase/types` | Don't hand-write DB row types |
| Icons | `lucide-react` | Don't import from other icon libs |
| Toast (imperative) | `@/hooks/use-toast` → `useToast()` | Don't import from `sonner` directly in components |
| Router | `react-router-dom` | — |
| Forms | `react-hook-form` + `@hookform/resolvers/zod` + `zod` | Don't use uncontrolled forms for validated inputs |
| Machine types | `@/types/machine` | Don't define machine enums locally |
| Connector layer | `@/connectors/jobline` | Components MUST NOT import from this — use hooks/store |
| Machine store | `@/store/machineStatusStore` | Only import in hooks, not in UI directly |

## 2. Styling Rules

| Rule | Detail |
|------|--------|
| **Use semantic tokens** | `bg-background`, `text-foreground`, `bg-primary`, `text-muted-foreground`, etc. |
| **Never hardcode colors** | ❌ `text-white`, `bg-black`, `bg-[#1a1a2e]` — use tokens |
| **All colors are HSL** | Defined in `src/index.css` `:root` / `.dark` blocks |
| **New colors** | Add to `index.css` as CSS vars → register in `tailwind.config.ts` |
| **Responsive** | Mobile-first. Use `sm:`, `md:`, `lg:` breakpoints |
| **Dark mode** | All tokens already support dark. Don't add `.dark:` overrides unless needed |

## 3. Component Structure

```
src/components/<domain>/ComponentName.tsx   ← component
src/components/<domain>/index.ts            ← barrel export (update it!)
src/hooks/use<Feature>.ts                   ← data/logic hook
src/components/<domain>/ComponentName.test.tsx ← test file
```

### Barrel Exports
When adding a component to a domain folder that has an `index.ts`, **always add the export** to that barrel file. Missing exports cause silent import failures.

## 4. Entitlement / Access Gating

| Audience | Gate With |
|----------|-----------|
| Paid/Enterprise feature | `<EntitlementGate feature="feature_key">` or `useEntitlements().canAccess()` |
| Role-restricted UI | Check `user_roles` via `useRoleArchitecture()` or `has_role()` |
| Org-scoped data | Always filter by `organization_id` in queries and RLS |
| Team-scoped data | Use `TeamContext` → `selectedTeam?.id` |

---

## 5. Data Layer Rules

| Rule | Detail |
|------|--------|
| **Never edit** `client.ts` or `types.ts` | Auto-generated by Lovable Cloud |
| **Never edit** `.env` or `supabase/config.toml` | Auto-managed |
| **DB changes** | Use migration tool only |
| **Queries** | Use `@tanstack/react-query` with `supabase` client |
| **Org context** | Use `useOrgContext()` from `OrgProvider` — never call `useUserOrganization()` independently in hooks/components |
| **Realtime** | `supabase.channel()` with **500ms debounced** `invalidateQueries()` — ensure RLS covers the table |
| **Polling** | Always set `refetchIntervalInBackground: false` to pause when tab is hidden |
| **Optimistic updates** | For user-facing state changes (status, priority), update local state immediately and rollback on error |
| **Error boundaries** | Wrap dashboard sections in `DashboardErrorBoundary` for graceful degradation |
| **Lazy loading** | Use `React.lazy()` + `<Suspense>` for components with heavy dependencies (Recharts, editors) |
| **Row limit** | Default 1000 rows — paginate or use `.range()` |

## 6. Testing Requirements

| Item | Required |
|------|----------|
| New hook with logic | ✅ Unit test |
| Bridge / transform function | ✅ Unit test |
| Store actions | ✅ Unit test |
| UI component (non-trivial) | ✅ Render test |
| Test location | Co-located: `ComponentName.test.tsx` or `hookName.test.ts` |
| Test framework | `vitest` + `@testing-library/react` |
| Mocks | Supabase mock at `src/test/mocks/supabase.ts` |

---

## 7. Pre-Commit Checklist

Before finalizing any component change, verify:

- [ ] Imports use `@/` aliases (no relative `../../`)
- [ ] No hardcoded colors — all semantic tokens
- [ ] Barrel `index.ts` updated if applicable
- [ ] Types come from `@/integrations/supabase/types` or `@/types/*`
- [ ] Supabase client from `@/integrations/supabase/client`
- [ ] Entitlement/role gate present if feature is restricted
- [ ] `organization_id` filter on all org-scoped queries
- [ ] Test file created or updated
- [ ] No direct connector imports in UI components
